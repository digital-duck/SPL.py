## 0. High-level Description
This project implements a research agent that utilizes an LLM to answer questions by first deciding whether to search the web for information or directly provide an answer. The agent operates through a series of steps: it receives a user query, prompts an LLM to determine the next action (search or answer), uses the search utility function (DuckDuckGo) to retrieve relevant information if searching is determined as the next action, then prompts the LLM again to synthesize a final answer based on the retrieved information and its existing knowledge. Finally, the agent outputs the generated answer. The entire process is managed by a flow of nodes, where each node performs a specific task, and the LLM drives the decision-making process based on the current state of the information.

## 1. Agent Overview
The agent is designed to answer questions by leveraging both web search and its own internal knowledge. It dynamically determines whether to search for information or directly respond to the query. The agent's key components are:
*   **Decision Node:** Determines whether to search or answer based on the LLM's response.
*   **Search Node:**  Utilizes DuckDuckGo to retrieve information from the web given a search query.
*   **Answer Node:**  Generates a final answer to the question, incorporating both the retrieved information and the LLM's knowledge.

## 2. Workflow
1.  **User Query:** The agent receives a user query.
2.  **Decision:** The Decision Node prompts the LLM to decide whether to search or answer.
3.  **Search (if applicable):** If the LLM determines a search is needed, the Search Node queries DuckDuckGo using the LLM's determined search query.
4.  **Answer Generation:** The Answer Node receives the search results (if any) and the original question, prompting the LLM to synthesize a final answer.
5.  **Output:** The agent outputs the generated answer.

## 3. Node Details

### 3.1 Decision Node
*   **Purpose:** Determines the next action - either search for information or provide an answer.
*   **Input:**  None.
*   **Output:**  A YAML string defining the next action ("search" or "answer") and any associated parameters.
*   **LLM Prompt:**  A complex prompt that instructs the LLM to analyze the current context, determine the most appropriate action, and output the decision in a structured YAML format.  The prompt explicitly guides the LLM to use a specific format for its response.

### 3.2 Search Node
*   **Purpose:** Performs a web search using DuckDuckGo.
*   **Input:**  A search query string.
*   **Output:**  A string containing the search results from DuckDuckGo.
*   **Implementation:** Calls the `search_web_duckduckgo` function.

### 3.3 Answer Node
*   **Purpose:** Generates a final answer to the question based on the available information.
*   **Input:** The original question and the retrieved information (if any).
*   **Output:** The final answer generated by the LLM.
*   **LLM Prompt:** A prompt that asks the LLM to synthesize a comprehensive answer based on the question and the given information.  It's crucial that this prompt instructs the LLM to use the retrieved information effectively.

## 4. Data Flow

The flow of data is as follows:
1.  User Query -> Decision Node
2.  Decision Node -> LLM (for decision) -> Decision Node (with action)
3.  Decision Node -> Search Node (if action is "search")
4.  Search Node -> DuckDuckGo -> Search Node (with search results)
5.  Search Node -> Answer Node (with search results)
6.  Answer Node -> LLM (for answer generation) -> Answer Node (with final answer)
7.  Answer Node -> Agent Output (final answer)

## 5. Technologies Used
*   **LLM:** GPT-4o (or another compatible LLM)
*   **Web Search:** DuckDuckGo API
*   **YAML Parsing:** Python's `yaml` library
*   **Request Library:** `requests` (for interacting with the DuckDuckGo API)
*   **DDGS:** DuckDuckGo Search library
