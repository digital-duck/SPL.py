PocketFlow is a 100-line minimalist LLM framework for building AI agents and pipelines. It is lightweight, zero bloat, zero dependencies, and zero vendor lock-in. Install with: pip install pocketflow. PocketFlow supports Node, BatchNode, Flow, and AsyncFlow primitives for composing multi-step LLM workflows.
SPL (Structured Prompt Language) is a declarative language for expressing LLM workflows. SPL 3.0 supports WORKFLOW, CREATE FUNCTION, GENERATE, CALL, WHILE, EVALUATE, RETURN, and EXCEPTION constructs. The spl3 CLI provides text2spl, mmd2spl, splc compile, and run commands for the full NDD (Natural-language Driven Development) round-trip.
Retrieval-Augmented Generation (RAG) is a technique that combines information retrieval with language model generation. A RAG pipeline first indexes a document corpus into a vector store, then at query time retrieves the most relevant chunks and passes them as context to the LLM. This grounds the answer in factual source material and reduces hallucination.
FAISS (Facebook AI Similarity Search) is an open-source library for efficient similarity search and clustering of dense vectors. It supports IndexFlatL2 (exact L2 search) and various approximate search indexes. FAISS is widely used for building scalable nearest-neighbor retrieval systems in RAG pipelines.
The NeurIPS-26 NDD experiment evaluates whether AI models can faithfully reconstruct a PocketFlow recipe from its natural-language description. The round-trip pipeline is: describe source code to spec (S1), convert spec to Mermaid diagram (S2), convert diagram to SPL (S3), compile SPL to target code (S4), re-describe the compiled code (S5), and measure semantic closure (S6).
Claude Sonnet 4.6 is Anthropic's mid-tier model in the Claude 4 family. It balances speed, cost, and capability. The claude_cli adapter in SPL.py allows calling Claude models directly via the Claude Code CLI, enabling tool use (WebSearch, Bash) within SPL GENERATE steps.
OpenAI text-embedding-ada-002 produces 1536-dimensional float32 vectors optimized for semantic similarity tasks. It is the default embedding model used in the pocketflow-rag recipe for both document indexing and query embedding. Cosine similarity or L2 distance can be used to rank retrieved chunks.
