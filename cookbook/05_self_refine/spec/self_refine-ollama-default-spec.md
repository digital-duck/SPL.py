## Summary

This implementation, called "Self-Refine Pattern", is a workflow for generating high-quality text through iterative refinement. It uses a combination of LLMs to improve the output based on feedback provided by another model. The workflow is designed to run multiple iterations, with each iteration refining the previous output until it meets certain criteria or reaches a maximum number of iterations.

## Detailed Specification

### 1. Purpose
The Self-Refine Pattern aims to create high-quality text through iterative refinement, using a combination of LLMs to improve the output based on feedback provided by another model.

### 2. High-level Description

This workflow consists of three main logical functions: `draft`, `critique`, and `refine`. The `draft` function generates an initial draft based on a prompt; the `critique` function provides feedback on the draft, indicating whether it meets the requirements or needs further refinement; the `refine` function improves the draft based on the provided feedback. The workflow runs multiple iterations of these functions, with each iteration refining the previous output until it meets certain criteria or reaches a maximum number of iterations.

The workflow uses two LLMs: one for generating initial drafts (`writer_model`) and another for providing feedback (`critic_model`). It also logs the progress and outputs to a directory. The final output is stored in a file named `final.md`.

### 4. Logical Functions / Prompts

| Name | Role in the workflow | Key prompt conventions (sentinel tokens, scoring, output format) |
| --- | --- | --- |
| `draft` | Generates initial draft | Prompt sent to writer_model; output stored in `@current` |
| `critique` | Provides feedback on draft | Feedback provided by critic_model; output stored in `@feedback` |
| `refine` | Improves draft based on feedback | Improved draft generated by refine(); output stored in `@current` |

### 5. Control Flow

1. Initial step: Generate initial draft using `draft`.
2. Loop condition: Check if iteration has reached the maximum number of iterations.
3. Branch logic:
	* If feedback indicates that the draft meets the requirements, return with status "complete" and update final output.
	* Otherwise, increment iteration and refine draft using `refine` and provide feedback to critic_model using `critique`.
4. Termination: Return with status "partial" if loop exhausted.
