## 0. High-level Description

This script implements a single-step multilingual translation workflow using SPL's `CREATE FUNCTION` and `PROMPT` constructs. A single `CREATE FUNCTION` named `greeting` defines the prompt template, accepting an `input TEXT` parameter (the user's message) and a `lang List` parameter (the target languages), with defaults of `'hello'` and a five-language list respectively; the prompt body instructs the LLM to act as a friendly assistant and translate the input into all languages in the list simultaneously. The workflow is expressed as a `PROMPT` block named `multilingual_greeting`, which binds `context.user_input` and `context.lang` to the function parameters and issues a `GENERATE greeting(input, lang)` call to produce the response. There is no iterative `WHILE` loop, no `EVALUATE` branching, no `CALL` side-effects, no `LOGGING`, and no `EXCEPTION` handling, making this a minimal, stateless, single-shot pattern suited for lightweight parametric prompting. The multi-language scope is driven entirely by the `lang` list parameter, allowing the caller to override target languages at invocation time via CLI flags. No explicit `RETURN` metadata (status codes, iteration counts) is emitted; the raw generated text is the direct result.

---

## 1. Purpose

Translate a user-supplied message into one or more target languages in a single LLM call, with both the message and the language list configurable at runtime.

---

## 2. Inputs

| Parameter | Default | Description |
|-----------|---------|-------------|
| `user_input` | *(none ā€” required at runtime)* | The text message the user wants translated, supplied via `context.user_input` |
| `lang` | `['Chinese', 'English', 'Spanish', 'French', 'German']` | Ordered list of target languages into which the input will be translated |

> Note: defaults for `input` and `lang` are declared inside `CREATE FUNCTION greeting`; `user_input` has no default and must be provided by the caller (e.g. `--adapter ollama user_input="hello wen"`).

---

## 3. Process

1. **Bind context** ā€” The `PROMPT multilingual_greeting` block reads `context.user_input` and `context.lang` from the runtime context (CLI arguments or calling environment).
2. **Set system role** ā€” `system_role('You are a friendly assistant.')` is applied to the LLM call.
3. **Resolve function parameters** ā€” `input` is set to `context.user_input`; `lang` is set to `context.lang` (falling back to the five-language default list if not supplied).
4. **Generate response** ā€” `GENERATE greeting(input, lang)` renders the `greeting` prompt template with the resolved parameters and sends it to the configured LLM adapter (e.g. Ollama).
5. **Return result** ā€” The LLM's translated output is returned directly as the workflow result.

## 4. Error Handling

*(No `EXCEPTION` blocks are defined in this script. All error handling is delegated to the runtime/adapter layer.)*

---

## 5. Output

The raw text generated by the LLM ā€” a single response containing the user's message translated into each of the requested languages. No structured `RETURN` metadata (status codes, iteration counts, or quality scores) is attached; the output is the plain generated string.
