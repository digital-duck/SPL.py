## Generated Code

```python
def merge_sorted(a, b):
    result = []
    i = j = 0
    while i < len(a) and j < len(b):
        if a[i] <= b[j]:
            result.append(a[i])
            i += 1
        else:
            result.append(b[j])
            j += 1
    result.extend(a[i:])
    result.extend(b[j:])
    return result
```

## Explanation

The function uses the classic two-pointer merge technique: it walks both lists simultaneously with index pointers `i` and `j`, always appending the smaller of the two current elements to the result and advancing that pointer forward. Because both inputs are already sorted, the smallest remaining element must be at one of the two current positions, so comparing just those two is always sufficient — no lookahead needed. This single pass through both lists gives O(n+m) time since each element is visited exactly once. The two `extend` calls at the end handle the edge case where one list is exhausted before the other, draining whichever has remaining elements in one shot (including the cases where one input is empty to begin with).
