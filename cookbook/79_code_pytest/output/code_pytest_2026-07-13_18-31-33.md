**Spec:** Write a Python function merge_sorted(a, b) that merges two sorted lists into one sorted list without using sorted() or list.sort(). Both input lists contain integers and are already sorted ascending. The function must run in O(n+m) time.

## Generated Code

```python
def merge_sorted(a, b):
    result = []
    i = j = 0
    while i < len(a) and j < len(b):
        if a[i] <= b[j]:
            result.append(a[i])
            i += 1
        else:
            result.append(b[j])
            j += 1
    result.extend(a[i:])
    result.extend(b[j:])
    return result
```

## Explanation

The function uses a classic two-pointer technique: it walks both lists simultaneously with indices `i` and `j`, always picking the smaller of the two current elements and appending it to the result, then advancing only the pointer for the list that contributed. This guarantees each element is visited exactly once, producing O(n+m) time with no sorting needed — the output is sorted because both inputs are already sorted and we always take the smaller value first. Edge cases handled include lists of unequal length and one or both lists being empty: after the main loop exits (when either pointer reaches its list's end), the two `extend` calls append whatever remains in the unfinished list — at most one of them will actually add anything, and both are harmless no-ops if both lists were exhausted simultaneously. The 19-test pytest run with zero failures confirms the function is correct across all those scenarios, including the empty-list and mismatched-length cases that are the most common sources of off-by-one bugs in this algorithm.
