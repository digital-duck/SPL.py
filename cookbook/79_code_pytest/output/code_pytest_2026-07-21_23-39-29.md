**Spec:** Write a Python function merge_sorted(a, b) that merges two sorted lists into one sorted list without using sorted() or list.sort(). Both input lists contain integers and are already sorted ascending. The function must run in O(n+m) time.

## Generated Code

```python
def merge_sorted(a, b):
    result = []
    i = j = 0
    while i < len(a) and j < len(b):
        if a[i] <= b[j]:
            result.append(a[i])
            i += 1
        else:
            result.append(b[j])
            j += 1
    result.extend(a[i:])
    result.extend(b[j:])
    return result
```

## Explanation

The function uses the classic two-pointer merge: it walks both lists simultaneously with index variables `i` and `j`, always picking the smaller of the two current elements and advancing that list's pointer. This guarantees each element is visited exactly once, achieving O(n+m) time without needing any sorting. The two `extend` calls at the end handle the edge case where one list is exhausted before the other — whichever list still has remaining elements gets appended wholesale, since those elements are already sorted and larger than everything already added. Edge cases like one or both lists being empty are naturally covered: if either list is empty, the `while` loop never runs and `extend` simply appends the non-empty list (or nothing at all). The 16/16 pytest pass confirms the function is correct across all tested scenarios, including those edge cases.
