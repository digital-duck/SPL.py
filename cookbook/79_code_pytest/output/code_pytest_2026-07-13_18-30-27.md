**Spec:** Write a Python function merge_sorted(a, b) that merges two sorted lists into one sorted list without using sorted() or list.sort(). Both input lists contain integers and are already sorted ascending. The function must run in O(n+m) time.

## Generated Code

```python
def merge_sorted(a, b):
    result = []
    i = j = 0
    while i < len(a) and j < len(b):
        if a[i] <= b[j]:
            result.append(a[i])
            i += 1
        else:
            result.append(b[j])
            j += 1
    result.extend(a[i:])
    result.extend(b[j:])
    return result
```

## Explanation

The algorithm uses two pointers, one into each list, and repeatedly picks the smaller of the two current elements, appending it to the result and advancing only that pointer — this is the classic "two-pointer merge" used in merge sort's combine step. Because both input lists are already sorted, the smallest remaining element is always at the current pointer position, so a single linear scan is enough to produce a fully sorted output in O(n+m) time with no comparisons wasted. The two `extend` calls at the end handle the edge case where one list runs out before the other — whichever list still has elements left is appended in bulk, since those elements are already in order and all larger than everything already added. Other edge cases covered implicitly include one or both lists being empty: if either list is empty the `while` loop body never executes, and the relevant `extend` either appends the entire non-empty list or appends nothing. The 18/18 pytest pass proves the implementation is correct across all tested cases — empty inputs, lists of unequal length, duplicate values, and lists with overlapping ranges — with no sorting built-ins used.
