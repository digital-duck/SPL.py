Dict

A dict (short for dictionary) is a fundamental data structure in computer science that stores mappings between keys and values. It allows for efficient lookups, insertions, and deletions of key-value pairs.

Definition

Formally, a dict can be defined as follows: let $D$ be a set of keys and $V$ be a set of values. A dict is a function $f: D \to V$ that assigns each key in $D$ to exactly one value in $V$. In other words, a dict is a binary relation between two sets that satisfies the following properties:

  • Each key is paired with exactly one value.
  • There are no duplicate keys.

    • Worked Example

    Consider a library catalog system where we want to store information about books. Each book has an ID (key) and title (value). We can represent this data using a dict as follows:

    library = {
    '123': 'Harry Potter',
    '456': 'The Lord of the Rings'
}

    In this example, the keys are the book IDs and the values are the corresponding titles.

    Suppose we want to add a new book with ID 789. We can do this by updating the dict:

    library['789'] = 'The Hunger Games'
print(library)  # Output: {'123': 'Harry Potter', '456': 'The Lord of the Rings', '789': 'The Hunger Games'}

    Key Theorem

    One of the key properties of dicts is that they support efficient lookups, insertions, and deletions. Specifically, for a dict $f: D \to V$, we have:

  • Look-up: If $k$ is an element of $D$, then there exists a unique value $v$ such that $(k,v) \in f$. In other words, if $k \in D$, then $f(k)$ returns the corresponding value.
  • Insertion: Given any key-value pair $(k,v)$, we can find an existing dict $f: D \to V$ and insert $(k,v)$ into it. More formally, given a set of keys $K$ and values $V$, we can construct a dict $f: K \to V$ that satisfies the following properties:

    • * For all $k \in K$, there exists a unique value $v$ such that $(k,v) \in f$. * For any key-value pair $(k,v)$, if $k \in K$, then $(k,v) \in f$.
