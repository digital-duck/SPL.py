## 0. High-level Description

This is a minimal "hello world" SPL program demonstrating the core pattern of a single-turn prompt dispatch. A single CREATE FUNCTION named `greeting` defines a prompt template with two parameter slots — `{user_input}` and `{lang}` — and carries an embedded description of SPL itself as static context within the prompt body. The PROMPT block named `hello_world` acts as the orchestration entry point, projecting fields from a `context` object (analogous to INPUT bindings) and wiring them into the `greeting` function via a GENERATE call. There is no WHILE loop, EVALUATE branch, EXCEPTION handling, or CALL side-effect — the design is intentionally flat and linear, making it a teaching artifact rather than a production workflow. Model selection is implicit (no INPUT parameter drives a model choice), and there is no explicit RETURN statement with metadata, meaning the output is the raw generated text from the single GENERATE invocation.

---

## 1. Purpose

Produce a single LLM-generated response to a user-supplied message, rendered in a chosen language, serving as a minimal runnable example of SPL syntax.

---

## 2. Inputs

| Parameter | Default | Description |
|---|---|---|
| `user_input` | `"Explain SPL"` | The text prompt or question submitted by the user |
| `lang` | `"English"` | The language in which the assistant should respond |

---

## 3. Process

1. The runtime resolves `context.user_input` and `context.lang` from the calling context (falling back to the defaults declared in `greeting` if absent).
2. The `greeting` prompt function is instantiated by substituting `{user_input}` and `{lang}` into its template body, which also injects a static paragraph explaining what SPL is.
3. A system role of `"You are a friendly assistant."` is prepended to the prompt via `system_role(...)`.
4. The assembled prompt is sent to the configured LLM via GENERATE.
5. The model's response text is returned as the output of the `hello_world` prompt block.

---

## 4. Error Handling

- *(None declared.)* The script contains no EXCEPTION clauses; any runtime or model errors will propagate unhandled to the caller.

---

## 5. Output

The raw TEXT response generated by the LLM for the `greeting` prompt. No explicit RETURN statement is present, so no structured metadata (status codes, iteration counts, etc.) is attached — the output is the model's reply verbatim.
