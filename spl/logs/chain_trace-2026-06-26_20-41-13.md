## Memory Cell

A memory cell is the fundamental building block of digital storage, designed to retain a single bit of binary information ($b \in \{0, 1\}$$) indefinitely until explicitly read or overwritten. Conceptually, it must implement non-volatile state retention—meaning its internal state remains stable despite fluctuations in input signals, provided the inputs are correctly managed. Physically, these cells are typically realized using clocked latches or flip-flops (FFs), which utilize cross-coupled feedback circuits, such as NAND or NOR gates. For instance, a basic Set-Reset (SR) Latch uses two intersecting pairs of inverters to achieve bistability: if the inputs are set ($S=1, R=0$), one inverter output is forced high, which then propagates through negative feedback to keep the other inverter’s input low, locking the cell into the 'Set' state.

**Worked Example:** Consider a positive-edge triggered D Flip-Flop (D FF), which samples the data $D$ at the rising edge of the clock $\text{CLK}$. The internal latch structure ensures that the output $Q_{next}$ is determined solely by the input $D$ during the active clock window, and this value is held until the next clock pulse. Mathematically, the state transition can be summarized by the equation:
$$Q(t+1) = D(t) \cdot \text{CLK}(t)$$
This means that if $\text{CLK}$ transitions from low to high at time $t$, the new value of $Q$ equals the value of $D$ sampled just before the transition. If $\text{CLK}$ is low or transitioning, the state $Q(t)$ remains unchanged (data retention).

## Pointer

In languages like C and C++, pointers are fundamental tools for implementing dynamic memory allocation, creating complex linked data structures, and achieving efficient parameter passing by reference. The primary syntax involves two operators: the address-of operator ($\&$), which returns the memory address of a variable, and the dereference operator ($\text{*}$), which accesses the value stored at the given address.

## Binary Tree

A binary tree is an ordered generalization of a tree data structure in which each node has at most two children, typically referred to as the *left child* and the *right child*. The structure imposes strict constraints: every node $v$ can have at most one left child and at most one right child. This constraint fundamentally limits the branching factor to two, making binary trees highly efficient for representing hierarchical data such as file systems or expression parsings. We denote a tree $T$ by its set of nodes $V(T)$ and its edge set $E(T)$.

For any subsequent node $v$, all keys in its left subtree must be less than $v$'s key, and all keys in its right subtree must be greater than $v$'s key. This ordering property is critical for efficient search operations.

## Heap

A heap is an abstract data structure that satisfies the heap property. Conceptually, it is often implemented using an array because of its inherent relationship to a nearly complete binary tree (or full binary tree for analysis purposes). A heap can be either a Max-heap or a Min-heap. The **Max-heap property** dictates that for every node $N$, the value of $N$ must be greater than or equal to the values of all its children. Conversely, in a Min-heap, the parent value must be less than or equal to the child values. Critically, while the heap property governs the relationship between parents and children, it does not impose any ordering constraint on siblings or cousins; only traversing from root to leaf via parent/child links maintains order. This structure guarantees that the maximum (or minimum) element is always located at the root node, making heap operations highly efficient for priority queue implementations.

Consider an array representation of a Max-heap storing the elements $\{10, 5, 20, 3, 1\}$. We want to insert a new value $x=15$ while maintaining the heap property. Since the structure is array-based, we place $x$ at the next available leaf node (index 5). The process then requires "bubbling up" or *heapifying up*. We compare 15 with its parent (20) and find that $15 < 20$, so no swap occurs. However, if we inserted $x=25$ at index 5, the comparison would be: $\{25\}$ vs Parent $\{3\}$. Since $25 > 3$, they swap. Now, $\{25\}$ is compared against its new parent (5). Since $25 > 5$, they swap again. This continues until $x$ reaches a node whose parent satisfies the Max-heap property, thus restoring the invariant.

### linked_list (19.4s)

## Linked List

A linked list is a fundamental linear data structure that organizes elements sequentially by having each element point to the next element in the sequence. Unlike an array, where memory addresses must be contiguous and the size must often be fixed at compile time, a linked list stores its nodes non-contiguously throughout memory. Each individual unit of storage, known as a *node*, typically encapsulates two parts: the actual data payload and one or more pointers (or references). For a singly linked list, each node $N_i$ contains $\text{data}_i$ and a pointer $\text{next\_ptr}_i$, which points to the memory address of the subsequent node. The entire structure is managed by maintaining a pointer to the initial element, known as the *Head*. If the Head pointer is $NULL$, the list is empty.

Consider a simple worked example: inserting a new node $N_{new}$ at the beginning (the head) of an existing list $L$. Suppose the current head of $L$ points to node $N_A$. To perform this insertion, we first set $N_{new}$'s pointer to point to $N_A$. We then update the overall Head pointer of the list to reference $N_{new}$. This entire operation requires only manipulating two pointers and does not require shifting any existing data elements, making it highly efficient.

The key theoretical advantage of linked lists lies in their dynamic nature regarding insertions and deletions. Given a pointer to a specific node within the structure, inserting or deleting that node can be accomplished in constant time $\mathcal{O}(1)$. This efficiency contrasts sharply with arrays, where insertion or deletion in the middle requires shifting potentially $N$ subsequent elements, leading to an $\mathcal{O}(N)$ time complexity. Specifically, if we traverse a list of length $L$, accessing the $k$-th element takes $\mathcal{O}(k)$ time, but modifying the pointers only requires constant-time pointer arithmetic, independent of the total size $N$.

## Queue

A queue is a linear data structure that adheres strictly to the First-In, First-Out (FIFO) principle. Conceptually, it mimics real-world waiting lines: the element that has been in the structure the longest is the next one to be removed. Operationally, a queue maintains two distinct ends: the *rear* (or tail), where new elements are added, and the *front* (or head), from which elements are removed. We define two primary operations: $\text{Enqueue}(x)$, which inserts element $x$ at the rear; and $\text{Dequeue}()$, which removes and returns the element currently at the front. Unlike stacks (which operate on LIFO principles), a queue's integrity relies entirely on maintaining this strict ordering, ensuring that data access is sequential relative to insertion time. Mathematically, if we consider an empty queue $Q$ and a sequence of operations, the state transitions must always respect the invariant: $\text{Front} \rightarrow \dots \rightarrow \text{Rear}$.

**Worked Example:** Consider initiating a queue $Q$ with capacity 5.
1. $\text{Enqueue}(A)$: $Q = [A]$ (A is both front and rear).
2. $\text{Enqueue}(B)$: $Q = [A, B]$.
3. $\text{Dequeue}()$: Returns $A$. $Q = [B]$. The element $A$ is removed from the front.
4. $\text{Enqueue}(C)$: $Q = [B, C]$.
5. $\text{Dequeue}()$: Returns $B$. $Q = [C]$.
This example demonstrates that even after multiple insertions and deletions, the removal order ($A$, then $B$) perfectly mirrors the initial insertion order $(A, B)$, confirming the FIFO invariant.

## Operating System Scheduler

The operating system scheduler is a core kernel component responsible for deciding which process or thread should utilize the Central Processing Unit (CPU) at any given time. Its primary function is to manage the CPU resource efficiently among competing processes, thereby maximizing throughput and ensuring fair utilization while meeting defined quality-of-service (QoS) requirements. Scheduling decisions are made based on various metrics, including process priority, required resources, and elapsed execution time. Formally, a scheduler $S$ operates by maintaining the ready queue $\mathcal{R}$ of processes awaiting CPU allocation and implementing a dispatch mechanism that selects a Process Control Block (PCB) from $\mathcal{R}$. A conceptual proof sketch for optimal scheduling involves demonstrating that any non-preemptive schedule sequence $\sigma = (P_1, P_2, \dots, P_n)$ can be improved by swapping the order of two adjacent processes $P_i$ and $P_{i+1}$ if running them in a different order reduces metrics like total waiting time or average turnaround time. The optimal choice must balance the competing goals of minimizing latency (responsiveness) and maximizing resource utilization (throughput).

Consider a simple CPU scheduling scenario involving three processes, $\mathcal{P}_1$, $\mathcal{P}_2$, and $\mathcal{P}_3$. Process $\mathcal{P}_1$ has an arrival time $A_1=0$ and a burst time $B_1=5$. Process $\mathcal{P}_2$ arrives at $A_2=1$ with $B_2=2$. Process $\mathcal{P}_3$ arrives at $A_3=2$ with $B_3=4$. If the scheduler implements First-Come, First-Served (FCFS), the execution order will be $(\mathcal{P}_1, \mathcal{P}_2, \mathcal{P}_3)$. The completion time for $\mathcal{P}_3$ would be $T_{completion} = 5 + 2 + 4 = 11$. In contrast, if a Shortest Job Next (SJN) scheduler were used and preemption was allowed, the order might change dynamically to minimize waiting times.

A key theoretical concept underlying scheduling performance is Little's Law, which states that for any stable system, the average number of items in a queue ($L$) equals the average arrival rate ($\lambda$) multiplied by the average time an item spends in the system ($W$): $L = \lambda W$. While Little's Law does not dictate the optimal scheduling algorithm, it provides a powerful tool for analyzing steady-state system performance and understanding the relationship between utilization and queue length.
