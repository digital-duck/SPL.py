### term (18.1s)

## Term

**Definition.** Let $\Sigma = (F, \text{ar})$ be a signature, where $F$ is a set of function symbols and $\text{ar}: F \to \mathbb{N}_0$ assigns each symbol its arity. Let $X$ be a countably infinite set of variables, disjoint from $F$. The set $\mathcal{T}(\Sigma, X)$ of *terms* over $\Sigma$ and $X$ is the smallest set satisfying: (i) $X \subseteq \mathcal{T}(\Sigma, X)$; (ii) if $f \in F$ with $\text{ar}(f) = n$ and $t_1, \ldots, t_n \in \mathcal{T}(\Sigma, X)$, then $f(t_1, \ldots, t_n) \in \mathcal{T}(\Sigma, X)$. Terms with no variables are called *ground terms*, forming the set $\mathcal{T}(\Sigma)$.

**Theorem (Term Algebra).** The structure $\mathcal{T}(\Sigma, X)$ equipped with the operations $f^{\mathcal{T}} : (t_1, \ldots, t_n) \mapsto f(t_1, \ldots, t_n)$ constitutes the *free $\Sigma$-algebra* over $X$. Concretely, for every $\Sigma$-algebra $\mathcal{A}$ and every assignment $\alpha: X \to A$, there exists a unique $\Sigma$-homomorphism $\hat{\alpha}: \mathcal{T}(\Sigma, X) \to \mathcal{A}$ extending $\alpha$.

**Proof.** Define $\hat{\alpha}$ by structural induction: $\hat{\alpha}(x) = \alpha(x)$ for $x \in X$, and $\hat{\alpha}(f(t_1, \ldots, t_n)) = f^{\mathcal{A}}(\hat{\alpha}(t_1), \ldots, \hat{\alpha}(t_n))$ for $f \in F$. Well-definedness follows from the inductive clause of the definition. Uniqueness is immediate, since any homomorphism extending $\alpha$ must satisfy the same inductive equations, and $\mathcal{T}(\Sigma, X)$ is generated by $X$. $\blacksquare$

**Remark.** This universality property identifies $\mathcal{T}(\Sigma, X)$ as the initial object in the category of $\Sigma$-algebras equipped with an $X$-assignment, situating the concept squarely within Birkhoff's equational logic \cite{Birkhoff1935} and Goguen–Meseguer's institution theory \cite{Goguen1992}. In term rewriting, $\mathcal{T}(\Sigma, X)$ carries a natural partial order via the subterm relation, and positions within terms are encoded as sequences in $\mathbb{N}^*$, enabling precise specification of rewrite rules as pairs $(l, r) \in \mathcal{T}(\Sigma, X)^2$ \cite{BaaderNipkow1998}.

### type (58.6s)

## Type

**Definition (Type).** A *type* $A$ in a formal system $\mathcal{T}$ is a syntactic entity that classifies terms: we write $a : A$ to assert that term $a$ *inhabits* type $A$. Formally, a type theory is given by a set of *formation rules* specifying when $A\ \mathsf{type}$ is derivable, *introduction rules* constructing canonical inhabitants, *elimination rules* prescribing how inhabitants may be consumed, and *computation rules* (or $\beta$/$\eta$-rules) governing definitional equality.

**Theorem (Curry–Howard Correspondence).** Let $\mathcal{T}$ be an intuitionistic natural-deduction system and $\lambda_{\to}$ the simply-typed $\lambda$-calculus. There exists a bijection
$$
\phi : \{\text{propositions of }\mathcal{T}\} \longleftrightarrow \{\text{types of }\lambda_{\to}\}
$$
under which derivations of a proposition $P$ correspond exactly to closed terms of type $\phi(P)$, and cut-elimination corresponds to $\beta$-reduction.

**Proof (Sketch).** The bijection is defined by induction on propositional structure. As the representative base case, implication $P \Rightarrow Q$ maps to the function type $A \to B$, with the $\to$-introduction rule corresponding exactly to $\lambda$-abstraction and the $\to$-elimination rule to application. One then verifies by structural induction on derivations that every natural-deduction rule translates to the corresponding $\lambda_{\to}$ typing rule, and that the normalization theorem for $\lambda_{\to}$ (strong normalization via Tait's reducibility argument) corresponds precisely to cut-elimination. $\square$

**Remark.** The correspondence generalizes to richer systems. Martin-Löf (1984) extends the bijection to first-order predicate logic via dependent types, and subsequent developments—including homotopy type theory (Univalent Foundations Program, 2013) and domain-theoretic denotational semantics à la Scott—further situate types within broader mathematical frameworks. These extensions are treated in subsequent sections.

### function_types (29.0s)

## Function Types

**Definition.** Let $A$ and $B$ be types in a simply-typed system $\lambda^\to$. The *function type* $A \to B$ (also written $B^A$ in categorical notation) is the type whose terms are total functions from $A$ to $B$. Formally, if $\Gamma, x : A \vdash e : B$ is derivable, then $\Gamma \vdash \lambda x.\, e : A \to B$ by the *abstraction* rule ($\lambda$-introduction). Conversely, the *application* rule ($\lambda$-elimination) states: if $\Gamma \vdash f : A \to B$ and $\Gamma \vdash a : A$, then $\Gamma \vdash f\, a : B$.

**Theorem** (Curry–Howard correspondence). In the simply-typed lambda calculus, the type $A \to B$ corresponds bijectively to the proposition $A \Rightarrow B$ under the Curry–Howard isomorphism: terms of type $A \to B$ are in natural bijection with proofs of the implication $A \Rightarrow B$ in intuitionistic propositional logic.

**Proof.** One constructs a functor $\Phi$ from the category of typing derivations to the category of intuitionistic proof trees. The abstraction rule maps exactly to the $\Rightarrow$-introduction rule, and application maps to $\Rightarrow$-elimination (modus ponens). $\beta$-reduction corresponds to proof normalisation; $\eta$-expansion corresponds to proof extensionality. The bijection is canonical and respects composition, establishing a category equivalence. $\square$

**Remark.** In categorical semantics, function types are precisely the *exponential objects* $B^A$ in a Cartesian closed category (CCC): an object equipped with a natural bijection $\mathrm{Hom}(C \times A,\, B) \cong \mathrm{Hom}(C,\, B^A)$ for all objects $C$. The existence of all such exponentials characterises CCCs (Lambek–Scott, *Introduction to Higher-Order Categorical Logic*, 1986), making this the canonical denotational model for $\lambda^\to$. The $A \to B$ construction is then the non-dependent base case of a broader hierarchy—generalising to dependent and resource-sensitive variants in richer type theories—but each such extension introduces its own primitive and is treated separately.
