## Comparison

In mathematics, comparison provides a powerful framework for analyzing unknown quantities by relating them to known ones. When dealing with sequences of real numbers $\{a_n\}_{n=1}^\infty$ or infinite series, comparison establishes bounds to determine properties such as convergence and divergence without direct calculation. Formally, two sequences $\{a_n\}$ and $\{b_n\}$ are compared using inequalities; for instance, if $0 \le a_n \le b_n$, we say that $\{a_n\}$ is bounded above by $\{b_n\}$. This technique allows us to infer properties of the smaller sequence from the larger one, provided certain limiting conditions are met. The underlying principle relies on the monotonic nature of convergence: if a quantity remains below a convergent ceiling, it must also converge.

## Memory Access

Memory access refers to the mechanism by which a computational unit, typically the Central Processing Unit (CPU), loads data from or stores data to locations within the main memory (RAM). At its core, successful memory access requires mapping a logical address generated by the CPU into a physical electrical signal that selects the correct storage location. This process is governed by the Memory Management Unit (MMU) and involves fundamental principles of addressing, timing, and latency. The primary challenge in efficient system design is minimizing the time delay between requesting data and retrieving it, as memory access is often the dominant bottleneck in modern computing architectures.

Consider a simple scenario where we wish to read the value stored at an address $A$. If the machine utilizes virtual memory, the CPU issues a virtual address $V$. The MMU must translate $V$ into a physical address $P$. This translation is typically done using page tables or segment registers. For example, if a system uses 4 KB pages and the virtual address space is 32 bits, the upper bits are used for the page number, which indexes into the page table, while the lower $\log_2(4096) = 12$ bits represent the offset within that page. The physical address $P$ is constructed by concatenating the translated page base address with the original offset: $P = (\text{Base Address}) \, \| \, \text{Offset}$.

**Worked Example:** A processor needs to access a specific variable, $\mathbf{x}$, located at virtual address $V = (0\text{xFF}00\text{A}8)$. Assume the system uses 4 KB pages and that the MMU translates the page number $(0\text{xFF}00)$ to the physical base address $(0\text{x}20000000)$. The offset is $0\text{xA}8$. The resulting physical address $P$ is calculated as:
$$P = (0\text{x}20000000) + (0\text{xA}8) = 0\text{x}20000\text{A}8$$
The memory controller then uses this physical address to activate the specific bank and row in RAM, retrieving the stored data.

**Key Theorem: Principle of Locality.** Memory access performance is significantly improved by exploiting temporal locality (the tendency to reuse recently accessed data) and spatial locality (the tendency to access data located near previously accessed data). Caching mechanisms—such as L1, L2, and L3 caches—are designed explicitly around this principle. They store copies of frequently used memory blocks closer to the CPU core to drastically reduce effective memory latency, thereby maximizing system throughput.

## BFS

Breadth-First Search (BFS) is a graph traversal algorithm used to explore all nodes in an arbitrary graph level by level. Given a connected graph $G = (V, E)$ and a designated starting node $s \in V$, BFS systematically explores the neighborhood of $s$ before moving on to the neighbors of those neighbors, and so forth. This process guarantees that nodes are visited in non-decreasing order of their distance from $s$. The algorithm typically utilizes a queue data structure (FIFO: First-In, First-Out) to manage the set of nodes to be processed, ensuring that all nodes at depth $k$ are fully explored before any node at depth $k+1$ is considered. To prevent infinite loops in graphs containing cycles and to ensure each node is processed only once, BFS maintains a set or array tracking visited nodes.

Consider a concrete graph $G=(V, E)$ where $V = \{A, B, C, D\}$ and the edges are $E = \{(A, B), (A, C), (B, D)\}$. If we initiate BFS starting at node $A$, the process proceeds as follows:
1. Initialize Queue $Q$: $[A]$. Mark $A$ as visited.
2. Dequeue $A$. Examine neighbors $(B, C)$. Enqueue unvisited neighbors: $Q = [B, C]$. Mark $B$ and $C$ as visited.
3. Dequeue $B$. Examine neighbor $D$. Enqueue unvisited neighbor: $Q = [C, D]$. Mark $D$ as visited.
4. Dequeue $C$. Examine neighbors (none unvisited). $Q = [D]$.
5. Dequeue $D$. Examine neighbors (none unvisited). $Q = []$.
The traversal completes. BFS successfully explored the graph in order: $A \to B, C \to D$.

**Key Theorem:** If a graph is represented by an adjacency list and BFS is initiated at node $s$, then the distance computed from $s$ to any reachable node $v$ using Breadth-First Search is guaranteed to be the shortest path length in terms of number of edges (i.e., it finds the minimum hop count). This property makes BFS ideal for solving unweighted shortest path problems, such as finding the closest facility or determining connectivity reachability.

## Recursion

Recursion is a fundamental concept in computer science and mathematics where a definition or algorithm relies on calls to itself. Formally, a recursive function solves a problem by breaking it down into smaller instances of the exact same problem. To ensure termination and prevent infinite loops, every well-defined recursive procedure must possess two components: a **base case** (the simplest instance that can be solved directly without further recursion) and a **recursive step** (where the function calls itself with modified input). Mathematically, if $P(n)$ is a property or calculation for an integer $n$, defining it recursively means expressing $P(n)$ in terms of $P(n-k)$ for some constant $k>0$.

Consider calculating the factorial of a non-negative integer $n$, denoted $n!$. The definition is naturally recursive: $n!$ is defined as $n \times (n-1)!$. This structure requires a base case. For $n=0$, the function must terminate, yielding $0! = 1$.

**Worked Example:** Calculate $4!$ using recursion.
The procedure is defined as:
$$
\text{Factorial}(n) = \begin{cases} 1 & \text{if } n = 0 \\ n \times \text{Factorial}(n-1) & \text{if } n > 0 \end{cases}
$$
Applying this to $4!$:
\begin{enumerate}
    \item $\text{Factorial}(4) = 4 \times \text{Factorial}(3)$
    \item $\text{Factorial}(3) = 3 \times \text{Factorial}(2)$
    \item $\text{Factorial}(2) = 2 \times \text{Factorial}(1)$
    \item $\text{Factorial}(1) = 1 \times \text{Factorial}(0)$
    \item $\text{Factorial}(0) = 1$ (Base Case reached, returns 1)
\end{enumerate}
Back-substituting the results yields $4 \times 3 \times 2 \times 1 \times 1 = 24$.

**Key Theorem: Master Theorem for Recurrence Relations.** For analyzing the time complexity of recursive algorithms that exhibit divide-and-conquer characteristics, we often rely on recurrence relations. The Master Theorem provides a cookbook method to solve recurrences of the form $T(n) = a T\left(\frac{n}{b}\right) + f(n)$, where $a \ge 1$, $b > 0$, and $f(n)$ is asymptotically positive. By comparing $f(n)$ against $n^{\log_b a}$, we can determine the asymptotic complexity $\mathcal{O}(\cdot)$. Specifically, if $f(n) = \Theta(n^{\log_b a})$, then $T(n) = \Theta(n^{\log_b a}) \log n$.

## Dynamic Programming

**Definition**
Dynamic Programming (DP) is a powerful algorithmic technique used primarily for solving complex problems that exhibit optimal substructure and overlapping subproblems. It operates on the principle of breaking down a large problem into smaller, interconnected subproblems. Crucially, instead of re-solving the same subproblem multiple times (as might happen in naive recursion), DP stores the results of these subproblems—often in a table or memoization structure—and refers back to these stored values when they are needed again. This approach transforms an exponential time complexity algorithm into a polynomial one. Mathematically, if $P$ is the original problem and $S_1, S_2, \dots, S_k$ are its subproblems, DP ensures that the solution for any subproblem $S_i$ is computed exactly once. This efficiency gain is fundamental to optimization problems in graph theory, combinatorial mathematics, and sequence alignment.

**Worked Example: Fibonacci Sequence**
Consider calculating the $n$-th Fibonacci number, $F(n)$, defined by the recurrence relation $F(n) = F(n-1) + F(n-2)$, with base cases $F(0)=0$ and $F(1)=1$. A purely recursive approach calculates $F(5)$ by recursively calculating $F(4)$ and $F(3)$. Notice that $F(3)$ will be calculated multiple times (once via $F(4)$ and once via the direct call to $F(3)$). Using DP, we build a memoization table (or bottom-up array) where we compute $F(2), F(3), \dots, F(n)$ sequentially. We define an array $DP[i]$ to store $F(i)$. The recurrence is then implemented iteratively:
$$DP[i] = DP[i-1] + DP[i-2], \quad \text{for } i \ge 2.$$
This iterative approach reduces the time complexity from $O(\phi^n)$ (exponential) to $O(n)$ (linear).

**Key Theorem: Optimal Substructure Property**
A problem possesses optimal substructure if an optimal solution to the overall problem can be constructed from optimal solutions to its subproblems. This property is necessary, but not sufficient, for DP. For example, in the shortest path problem on a Directed Acyclic Graph (DAG), finding the shortest path from $u$ to $v$ requires knowing that the subpath from $u$ to any intermediate node $w$ must itself be the shortest possible path between those two points. This principle allows us to use Bellman-Ford or Dijkstra's algorithm, which are classic DP applications, because the overall optimal solution is built piece by local optimal decisions.

## Greedy

A greedy algorithm is an algorithmic paradigm that builds a solution step-by-step, always making the locally optimal choice at each stage with the hope of reaching a globally optimal solution. Unlike dynamic programming, which computes solutions for overlapping subproblems and combines results to guarantee optimality, the greedy approach makes decisions purely based on immediate gain without considering future consequences or potential trade-offs. The defining characteristic is that once a choice is made, it cannot be revisited or altered. This simplicity often leads to highly efficient time complexity, making greedy algorithms attractive when they are applicable. However, it is crucial to understand that the local optimum does not always guarantee the global optimum; if the underlying problem structure lacks the necessary properties (such as the optimal substructure property satisfied by dynamic programming), the algorithm may fail entirely.

Consider the classic Coin Change Problem. Given a set of coin denominations $D = \{d_1, d_2, \dots, d_k\}$ and a target amount $A$, find the minimum number of coins required to sum up to $A$. A greedy strategy involves repeatedly selecting the largest denomination coin that is less than or equal to the remaining amount until the total reaches $A$.

**Worked Example:** Suppose we need to make change for 63 cents using standard US denominations: $\{1, 5, 10, 25\}$.
*   **Step 1 (Amount=63):** The largest usable coin is 25. Selection: 25. Remaining amount: $63 - 25 = 38$.
*   **Step 2 (Amount=38):** Largest usable coin is 25. Selection: 25. Remaining amount: $38 - 25 = 13$.
*   **Step 3 (Amount=13):** Largest usable coin is 10. Selection: 10. Remaining amount: $13 - 10 = 3$.
*   **Step 4 (Amount=3):** Largest usable coin is 1. Selection: 1. Remaining amount: 2.
*   **Step 5:** Select 1. Remaining amount: 1.
*   **Step 6:** Select 1. Remaining amount: 0.
The greedy choice yields the solution $\{25, 25, 10, 1, 1\}$, requiring 5 coins. In this specific case (standard currency systems), the greedy approach is provably optimal.

## Route Planning

Route planning is a domain within graph theory and operations research focused on finding an optimal path between two or more specified nodes (vertices) in a weighted graph. Mathematically, the problem seeks to minimize a cost function—which could represent distance, time, fuel consumption, or energy expenditure—while traversing edges that connect the vertices. A formal representation involves defining a graph $G = (V, E)$, where $V$ is the set of locations and $E$ is the set of traversable connections. Each edge $(u, v) \in E$ is assigned a non-negative weight $w(u, v)$, representing the cost of traversing that connection. The goal of route planning is to determine a sequence of vertices $P = (v_0, v_1, \dots, v_k)$ such that $\sum_{i=0}^{k-1} w(v_i, v_{i+1})$ is minimized, subject to constraints such as capacity limits or mandatory waypoints.

Consider a simplified scenario involving finding the shortest path in a road network modeled as an undirected graph $G$. Let $A$ and $B$ be two cities (vertices), and the edges represent roads with weights equal to driving time. We seek the minimum total weight path from $A$ to $B$. If we use Dijkstra's algorithm, we maintain a distance array $d[v]$ initialized to $\infty$, except for the starting vertex $d[A] = 0$. At each step, we select the unvisited vertex with the smallest tentative distance and relax its neighbors, updating their distances if a shorter path is found. This systematic relaxation guarantees that when a vertex is finalized, the shortest path from $A$ to it has been determined.

The foundational principle governing efficient route planning in graphs with non-negative edge weights is Dijkstra's Algorithm. The core assumption underlying this algorithm is the *non-negativity* of edge weights; if negative weights were permitted, cycles could lead to arbitrarily short paths, requiring alternative methods like the Bellman-Ford algorithm or SPFA. A proof sketch for Dijkstra’s relies on the principle of optimality: if the shortest path from $A$ to $B$ passes through an intermediate vertex $X$, then the segment from $A$ to $X$ must itself be the shortest path from $A$ to $X$. This property allows greedy selection of the closest unvisited node.
