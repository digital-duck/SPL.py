# TOOL_API Function Registry — cookbook-pocketflow

Quick-lookup table of all `CREATE TOOL_API` functions defined across migrated recipes.
Use this when writing tools.spl for a new recipe to avoid duplication and to identify
stdlib promotion candidates.

**Columns:**
- **Source** — `tools.spl` (hand-written S4) or `inline` (auto-generated by S3 inside the main .spl)
- **Stdlib?** — `yes` already in stdlib · `candidate` worth promoting · `recipe-specific` keep local

## Stdlib Promotion Shortlist

Priority order based on reuse potential across future recipes:

| Priority | Function | Target | Rationale |
|----------|----------|--------|-----------|
| ⭐⭐⭐ | `parse_yaml` | stdlib (Python) | YAML parsing is needed in almost every agent recipe that calls tools returning structured data |
| ⭐⭐⭐ | `extract_yaml_field` | stdlib (Python) | Companion to `parse_yaml`; replaces ad-hoc LLM field extraction |
| ⭐⭐⭐ | `append_turn` | stdlib (Python) | Universal chat history helper; pairs with existing `trim_turns` |
| ⭐⭐ | `validate_yaml` | stdlib (Python) | Common quality gate in agentic loops |
| ⭐⭐ | `chunk_documents` | `stdlib_rag.spl` | Needed for any RAG recipe |
| ⭐⭐ | `embed_documents` + `embed_query` + `create_faiss_index` + `retrieve_document` | `stdlib_rag.spl` | Promote as a unit |
| ⭐ | `get_section_by_index` | drop → use `list_get` | Already covered by stdlib |
| ⭐ | `search_web` (in 004_agent/tools.spl) | drop → use stdlib | Already in stdlib; remove from tools.spl |
