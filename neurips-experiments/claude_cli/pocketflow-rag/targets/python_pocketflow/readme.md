# pocketflow-rag (Deterministic Python/PocketFlow)

Generated by `splc --deterministic --lang python/pocketflow` from `pocketflow-rag.spl`.

Pattern: **linear** — sequential GENERATE/CALL chain, one Node per step.

## Install

```bash
pip install pocketflow click
```

## Run

```bash
python pocketflow-rag_python_pocketflow.py \
  --texts "Machine learning is a subset of AI..." \
  --query "What is machine learning?"
```

## Pipeline Structure

The `rag_e2e` workflow maps to three nodes:

| Step | Node | SPL | Stores |
|------|------|-----|--------|
| 1 | `Step1_Rag_indexNode` | `CALL rag_index(@texts)` | `@index` |
| 2 | `Step2_Chunk_documentsNode` | `GENERATE chunk_documents(@texts)` | `@chunks` |
| 3 | `Step3_Rag_queryNode` | `CALL rag_query(@query, @index, @chunks)` | `@answer` |

## ETL Mapping

| PocketFlow | ETL role | SPL construct |
|------------|----------|---------------|
| `prep(shared)` | Extract | read `@variables` from shared store |
| `exec(prep_res)` | Transform | `GENERATE` / LLM call |
| `post(shared, prep_res, exec_res)` | Load | write results back + return action |
| `shared` dict | Staging area | SPL `@variable` scope |
| `Flow(start=...)` | Pipeline DAG | `WORKFLOW` + control flow |

## Known Limitation: CALL stubs

`CALL` statements (sub-workflow invocations) are compiled as **LLM simulation stubs**:

```python
return _call_llm("llama3.2", f"Execute rag_index: {texts}")
```

The stub passes only the raw input to the LLM — it does **not** execute the
actual sub-workflow logic (chunking → embedding → FAISS indexing). As a result:

- The LLM may answer any query from its parametric knowledge, regardless of
  the provided `--texts`.
- A query about an unrelated topic (e.g. "What is interferometry?") will
  return a factually correct answer from the model's weights, **not** a
  retrieval failure.

**This is structural fidelity, not behavioral fidelity.** The pipeline shape
is correct; the stub nodes are placeholders for production wiring.

### Production replacement

Replace each CALL stub with a direct Python function call that implements the
sub-workflow:

```python
# Step 1 — replace stub with real rag_index implementation
def exec(self, prep_res):
    texts = prep_res
    chunks = chunk_documents(texts)          # real chunker
    embeddings = embed_documents(chunks)     # real embedder
    index = create_faiss_index(embeddings)   # real FAISS index
    return index
```
