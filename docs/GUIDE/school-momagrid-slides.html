What is Momagrid?

Momagrid 是什么？

🖥️ Hub (1 machine)

  • Receives AI task requests
  • Routes tasks to available agents
  • Tracks results and timing
  • One command: mg hub up

💻 Agent (every machine)

  • Runs Ollama locally
  • Picks up tasks from the Hub
  • Returns results to the caller
  • One command: mg join <url>

SPL is the workflow language — write AI pipelines like SQL, run them on the grid.

How It Works

工作原理

Just Two Commands

只需两条命令

🖥️ Hub machine (teacher / one student)
mg hub up
Starts the grid. Share the IP shown with the class.
💻 Every other machine (students)
mg join http://192.168.x.x:9000
That's it — this machine is now part of the grid.

Verify with mg agents — see every student's machine appear in real time.

Write AI Workflows in SPL

用SPL编写AI工作流 — 像SQL一样简洁

-- A self-refining essay writer (35 lines)
WORKFLOW self_refine
    INPUT:  @topic TEXT
    OUTPUT: @essay TEXT
DO
    GENERATE writer(@topic)             INTO @draft
    CALL     critique_workflow(@draft)  INTO @feedback
    GENERATE refiner(@draft, @feedback) INTO @essay
    COMMIT   @essay
END
# Run on the school grid — zero .spl changes
spl3 run self_refine.spl --adapter momagrid \
     --param topic="Why is the sky blue?"

Each GENERATE is dispatched to an available agent. More students online = faster execution.

Setup in 1 Hour

1小时内完成部署

⏱ 0–20 min: Prerequisites

# Every machine (Hub + agents)
ollama pull llama3.2   # ~2 GB

# Install mg CLI
git clone .../momagrid
cd momagrid && go build -o mg .
ln -s $(pwd)/mg ~/.local/bin/mg

⏱ 20–40 min: Launch the Grid

# Hub machine (teacher's)
mg hub up
# → Hub at http://192.168.x.x:9000

# Every student machine
mg join http://192.168.x.x:9000

# Verify
mg agents   # all students appear

⏱ 40–60 min: First Workflow

pip install spl3
spl3 run hello_world.spl \
     --adapter momagrid
mg tasks   # watch it fly!
